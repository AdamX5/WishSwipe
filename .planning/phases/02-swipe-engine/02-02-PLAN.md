useSprings + useDrag canonical pattern with topIndex (REQUIRED — resolves display vs queue index ambiguity):
```typescript
import { useSprings, animated, to as interpolate } from '@react-spring/web'
import { useDrag } from '@use-gesture/react'

const SPRING_COUNT = 20          // fixed — hooks cannot have dynamic counts
const SWIPE_VELOCITY_THRESHOLD = 0.2
const SWIPE_DISTANCE_THRESHOLD = 100

const to = (i: number) => ({ x: 0, y: i * -4, scale: 1, rot: 0 })
const from = () => ({ x: 0, rot: 0, scale: 1.5, y: -1000 })
const trans = (r: number, s: number) => `rotateZ(${r}deg) scale(${s})`

// topIndex = absolute index into queue[] pointing at the current top card
// gone = Set of absolute queue indices that have been swiped off
const topIndex = useRef(0)
const gone = useRef(new Set<number>())

const [springs, api] = useSprings(SPRING_COUNT, i => ({ ...to(i), from: from() }))

// bind(absoluteIndex) — pass absolute queue index so gone.current uses consistent keys
const bind = useDrag(({ args: [index], active, movement: [mx], direction: [xDir], velocity: [vx], last }) => {
  const trigger = vx > SWIPE_VELOCITY_THRESHOLD || Math.abs(mx) > SWIPE_DISTANCE_THRESHOLD
  const dir = xDir < 0 ? -1 : 1

  if (last && trigger) {
    gone.current.add(index)           // index is the absolute queue index
    topIndex.current += 1             // advance the top pointer
    const swipeDir = dir === 1 ? 'right' : 'left'
    const product = queue[index]
    if (product) {
      recordSwipeMutation({ ... }).catch(console.error)
    }
  }

  api.start(i => {
    if (index !== i) return           // only animate the spring slot for this card
    const isGone = gone.current.has(index)
    return {
      x: isGone ? (200 + window.innerWidth) * dir : active ? mx : 0,
      rot: mx / 100 + (isGone ? dir * 10 * vx : 0),
      scale: active ? 1.1 : 1,
      config: { friction: 50, tension: active ? 800 : isGone ? 200 : 500 },
    }
  })
})

// Render: display window = queue[topIndex..topIndex+3]
// Each card at display position i uses absolute queue index: topIndex.current + i
// Spring slot for card at absolute index N is also N (1:1 mapping, stable across deck advances)
const displayQueue = queue.slice(topIndex.current, topIndex.current + 3)

return (
  <div className="relative flex items-center justify-center" style={{ height: '24rem' }}>
    {queue.length === 0 || topIndex.current >= queue.length ? (
      <p className="text-gray-400">No more products — check back later</p>
    ) : (
      displayQueue.map((product, i) => {
        const absIndex = topIndex.current + i     // absolute queue index = spring slot index
        const { x, y, rot, scale } = springs[absIndex]
        return (
          <animated.div
            key={product._id}
            style={{ x, y, position: 'absolute', zIndex: displayQueue.length - i }}
          >
            <animated.div
              {...bind(absIndex)}               // pass absolute index — NOT display position i
              style={{ transform: interpolate([rot, scale], trans), touchAction: 'none' }}
            >
              <SwipeCard product={product} xSpring={x} />
            </animated.div>
          </animated.div>
        )
      })
    )}
    {topIndex.current < queue.length && displayQueue.length < 3 && (
      <div
        className="absolute rounded-2xl bg-gray-100"
        style={{ width: '18rem', height: '24rem', zIndex: 0 }}
      />
    )}
  </div>
)
```

**Why topIndex is required:** If display position `i` (0-2) is passed to `bind()` and stored in `gone.current`, then after the top card is swiped off, the next card inherits display position 0. Since `gone.current` still contains 0, the new top card is immediately treated as gone and flies off without user input. Using absolute indices (`topIndex.current + i`) means each card has a unique, stable key in `gone.current` for its entire lifetime in the queue.

**Index strategy (critical — must use absolute queue indices throughout):**

The deck uses two index spaces. Confusing them causes the card-advance bug where the next card immediately flies off screen:
- **Display position `i`** (0, 1, 2): which slot in the 3-card visual stack a card occupies right now
- **Absolute queue index** (`topIndex.current + i`): that card's stable position in the full `queue[]` array

`gone.current` and `springs` MUST use absolute queue indices only. `bind()` MUST receive the absolute queue index. Display position `i` is only used for `zIndex` ordering in the render.

**Render — 3-card display window using absolute indices:**

`displayQueue` is a window into `queue[]` starting at `topIndex.current`. Each card at display position `i` has absolute queue index `topIndex.current + i`. That absolute index is used for: the spring slot lookup, the `bind()` argument, and `key`.

```tsx
const displayQueue = queue.slice(topIndex.current, topIndex.current + 3)
const exhausted = queue.length > 0 && topIndex.current >= queue.length

return (
  <div className="relative flex items-center justify-center" style={{ height: '24rem' }}>
    {queue.length === 0 || exhausted ? (
      <p className="text-gray-400">No more products — check back later</p>
    ) : (
      displayQueue.map((product, i) => {
        const absIndex = topIndex.current + i   // stable identity for this card
        const { x, y, rot, scale } = springs[absIndex]
        return (
          <animated.div
            key={product._id}
            style={{ x, y, position: 'absolute', zIndex: displayQueue.length - i }}
          >
            <animated.div
              {...bind(absIndex)}               // MUST pass absIndex, not display position i
              style={{ transform: interpolate([rot, scale], trans), touchAction: 'none' }}
            >
              <SwipeCard product={product} xSpring={x} />
            </animated.div>
          </animated.div>
        )
      })
    )}
    {!exhausted && displayQueue.length < 3 && (

    )}
  </div>
)
```

Do NOT use `queue.slice(0, 3)` — that always shows the first three products and does not advance as cards are swiped off.

Do NOT compute a `springIndex` variable from the display position that is then discarded — the only index used is `absIndex`.

Add `touchAction: 'none'` on the draggable `animated.div` to prevent browser scroll interference on mobile.
