**2. Create `app/swipe/_components/SwipeOverlay.tsx`:**

This is a pure presentational component driven entirely by the `xSpring` SpringValue — no React state, no event handlers.

```tsx
'use client'
import { animated } from '@react-spring/web'
import type { SpringValue } from '@react-spring/web'

interface SwipeOverlayProps {
  xSpring: SpringValue<number>
}

export default function SwipeOverlay({ xSpring }: SwipeOverlayProps) {
  return (
    <animated.div
      style={{
        position: 'absolute',
        inset: 0,
        borderRadius: 'inherit',
        pointerEvents: 'none',
        display: 'flex',
        alignItems: 'center',
        justifyContent: 'center',
        backgroundColor: xSpring.to(x =>
          x > 0
            ? `rgba(0, 200, 0, ${Math.min(Math.abs(x) / 200, 0.35)})`
            : `rgba(200, 0, 0, ${Math.min(Math.abs(x) / 200, 0.35)})`
        ),
      }}
    >
      {/* Heart icon — visible when dragging right */}
      <animated.span
        style={{
          fontSize: '4rem',
          opacity: xSpring.to(x => (x > 20 ? Math.min((x - 20) / 80, 1) : 0)),
          position: 'absolute',
        }}
        aria-hidden="true"
      >
        ❤️
      </animated.span>

      {/* X icon — visible when dragging left */}
      <animated.span
        style={{
          fontSize: '4rem',
          opacity: xSpring.to(x => (x < -20 ? Math.min((Math.abs(x) - 20) / 80, 1) : 0)),
          position: 'absolute',
        }}
        aria-hidden="true"
      >
        ✕
      </animated.span>
    </animated.div>
  )
}
```

f) **Undo handler** — defined inside SwipeDeck, passed as a prop to SwipeShell or exported via a callback prop:
```typescript
const handleUndo = async () => {
  const lastSwipe = await undoSwipeMutation()
  if (!lastSwipe) return

  // Find the last swiped card index and restore it
  const lastGoneIndex = Math.max(...Array.from(gone.current))
  if (lastGoneIndex >= 0) {
    gone.current.delete(lastGoneIndex)
    api.start(i => {
      if (i !== lastGoneIndex) return
      return { x: 0, rot: 0, scale: 1, config: { tension: 500, friction: 50 } }
    })
  }
}
```

g) Expose `handleUndo` to the parent: SwipeDeck should accept an optional `onUndo?: (handler: () => void) => void` prop via `useEffect` to register it, OR (simpler) return the undo handler via a `ref` exposed to the parent. Simplest approach: add an `undoRef` passed from SwipeDeck up — but since the page is a server component, the cleanest approach is to move the Undo button INTO SwipeDeck itself, rendering it below the deck or using an absolute-positioned button.

**Simplest correct approach:** Add the Undo button directly inside SwipeDeck's return JSX, below the card stack:
```tsx
<div className="flex justify-center mt-6">

    onClick={handleUndo}
    disabled={gone.current.size === 0}
    className="px-6 py-2 rounded-full bg-white shadow-md border border-gray-200 text-gray-700 font-medium hover:bg-gray-50 disabled:opacity-40 disabled:cursor-not-allowed"
```

**Note on `disabled` and `gone.current`:** `gone.current.size === 0` is a ref, so it doesn't trigger re-renders. The disabled state won't update reactively. For v1, this is acceptable — the button disables on initial mount and re-enables appearance after any swipe causes a re-render. For a perfect disabled state, track `goneCount` in state alongside the ref; add `const [goneCount, setGoneCount] = useState(0)` and increment it when a card goes, then use `goneCount === 0` for the disabled condition.

Implement the `goneCount` state approach (not just the ref) so the Undo button correctly reflects availability without waiting for an unrelated re-render.
